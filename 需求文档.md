## 🚀 PSD to HTML 自动化转换工具需求文档

### 1. 项目目标
实现一个 Node.js 脚本，解析指定的 `.psd` 文件，并自动生成对应的静态 HTML 页面、图片资源及字体元数据。

### 2. 技术栈要求
*   **语言**: Node.js (Esm/Typescript)
*   **PSD 解析**: `ag-psd` (支持解析图层属性及提取 Canvas)
*   **图像处理**: `sharp` (用于将图层 Canvas 转换为高质量的 WebP/PNG)
*   **文件操作**: `fs-extra`

### 3. 核心功能点 (Core Requirements)

#### A. 资源解析与导出
*   **图层转图片**: 
    *   识别非文本图层（或带有特定前缀的图层），利用 `ag-psd` 提取其像素数据。
    *   使用 `sharp` 将提取的图层导出为 `images/layer_${index}.png`。
    *   **优化**: 自动跳过不可见图层，并处理图层的 `opacity`。
*   **字体元数据导出**: 
    *   扫描所有文本图层，提取 `fontName` (PostScript 名称)、`fontSize`、`fillColor`。
    *   去重后输出一份 `fonts_usage.json`，包含本项目用到的所有字体列表，方便后续补全 webfont。

#### B. HTML/CSS 还原逻辑
*   **布局方案**: 采用 `absolute` 绝对定位布局。
*   **容器设置**: 外层 `div` 容器宽高必须等于 PSD 画布宽高，设置 `position: relative`。
*   **文本处理**: 
    *   保留文本内容、颜色、字号、对齐方式。
    *   将 PSD 文本样式映射为标准的 CSS 属性（如 `font-family`, `font-size`, `color`, `line-height`）。
*   **图层映射**: 
    *   每个图层对应一个 `div` 或 `img`。
    *   利用图层的 `left`、`top`、`width`、`height` 属性生成内联样式。

#### C. 输出结构
转换完成后，目录结构应如下：
```text
/dist
  ├── index.html         # 还原后的静态页面
  ├── fonts_usage.json   # 字体信息统计
  └── images/            # 导出的图层切图
      ├── layer_1.png
      └── ...
```

### 4. 开发者提示 (For Claude Code)
1.  **处理嵌套**: PSD 是树状结构，请使用递归函数处理 `children` 数组。
2.  **坐标转换**: 注意 `ag-psd` 提供的坐标是相对于画布的绝对位置。
3.  **异常处理**: 针对大型 PSD，建议增加内存限制提示，并处理可能存在的空图层。
4.  **合并逻辑**: 如果图层被标记为“已合并”，则直接作为整张图片导出。
